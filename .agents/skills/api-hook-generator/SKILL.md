# Skill: API Hook Generator

## ℹ️ Objective
For every unique `data_ref` referenced in a processed UI Manifest, generate a typed React Query hook stub file. A `data_ref` is one of two types:
- **REST operationId**: a non-deprecated `operationId` from `docs/02-design-specs/api-contracts/openapi.yaml` → generates a REST hook at `src/hooks/use-{operationId}.ts`.
- **GraphQL resolver name**: a method name from a `<<GraphQLResolver>>` interface in `docs/02-design-specs/uml/*_contract.puml` (e.g., `listGradeRecords`, `listClasses`) → generates a GraphQL hook at `src/hooks/use-{resolverName}.ts`.

> ⚠️ Collection GET operationIds in openapi.yaml are marked `deprecated: true` with `DEPRECATED_TO_GRAPHQL`. When a `data_ref` matches a deprecated REST operationId, treat it as a **GraphQL resolver name** and use the GraphQL hook template.

---

## 🛠️ Generator Rules

### 1. Data Ref Type Resolution
For each `data_ref` value collected from the manifest:
1. Look it up in `docs/02-design-specs/api-contracts/openapi.yaml`.
   - If found **and NOT deprecated** → **REST path**. Extract HTTP method, path, parameters, and 200 response schema. Use the REST hook template (§3).
   - If found **and `deprecated: true`** → **GraphQL path**. Treat the value as a GraphQL resolver method name. Look up the resolver in `docs/02-design-specs/uml/*_contract.puml`. Use the GraphQL hook template (§3b).
   - If NOT found in openapi.yaml → **GraphQL path**. Look up in `docs/02-design-specs/uml/*_contract.puml`. Use the GraphQL hook template (§3b).
2. **Skip** any `data_ref` that equals `FIXME_DATA_REF` — log a warning in the report.

### 2. Naming Conventions
| Source | Derived Name |
|---|---|
| `data_ref` (camelCase) | File: `use-{dataRef}.ts` (kebab-case) |
| Response entity name | `{EntityName}` (PascalCase) |
| Hook function | `use{DataRef}` (PascalCase prefix) |
| Query key const | `{dataRef}Keys` |

### 3. REST Hook Template
Use when `data_ref` resolves to a non-deprecated REST `operationId`:

```typescript
// AUTO-GENERATED by Frontend Engineer workflow
// Source: docs/02-design-specs/api-contracts/openapi.yaml (operationId: {operationId})
// DO NOT EDIT MANUALLY — expand types from OpenAPI response schema

import { useQuery } from '@tanstack/react-query';
import { api } from '@/lib/api-client';

// ── Response Type (expand from OpenAPI 200 response schema) ───────────────
export interface {ResponseTypeName} {
  // TODO: expand fields from openapi.yaml components/schemas/{SchemaName}
  [key: string]: unknown;
}

// ── Query Key Factory ──────────────────────────────────────────────────────
export const {operationId}Keys = {
  all: ['{operationId}'] as const,
  detail: (id: string) => ['{operationId}', id] as const,
};

// ── Hook ───────────────────────────────────────────────────────────────────
export function use{HookName}(params?: Record<string, unknown>) {
  return useQuery({
    queryKey: {operationId}Keys.all,
    queryFn: () => api.{httpMethod}<{ResponseTypeName}>('{path}', { params }),
  });
}
```

### 3b. GraphQL Hook Template
Use when `data_ref` is a GraphQL resolver method name (deprecated REST operationId or resolver from `*_contract.puml`).
Extract the resolver's return type and filter input from the `<<GraphQLResolver>>` interface in `docs/02-design-specs/uml/*_contract.puml`.

```typescript
// AUTO-GENERATED by Frontend Engineer workflow
// Source: docs/02-design-specs/uml/{entity}_contract.puml (resolver: {resolverMethodName})
// DO NOT EDIT MANUALLY — expand query fields from interface contract return type

import { useQuery } from '@tanstack/react-query';
import { gql, request } from 'graphql-request';
import { GRAPHQL_ENDPOINT } from '@/lib/graphql-client';

// ── Response Type (expand from resolver return type in contract) ───────────
export interface {ResponseTypeName} {
  // TODO: expand fields from {EntityName} domain entity
  [key: string]: unknown;
}

// ── GraphQL Query Document ─────────────────────────────────────────────────
// NOTE: Stub only — expand field selection to match what the component needs.
const {RESOLVER_CONST}_QUERY = gql`
  query {resolverMethodName}($filter: {FilterInputType}) {
    {resolverMethodName}(filter: $filter) {
      # TODO: expand field selection from entity schema
    }
  }
`;

// ── Query Key Factory ──────────────────────────────────────────────────────
export const {resolverMethodName}Keys = {
  all: ['{resolverMethodName}'] as const,
  filtered: (filter: Record<string, unknown>) =>
    ['{resolverMethodName}', filter] as const,
};

// ── Hook ───────────────────────────────────────────────────────────────────
export function use{HookName}(filter?: Record<string, unknown>) {
  return useQuery({
    queryKey: filter
      ? {resolverMethodName}Keys.filtered(filter)
      : {resolverMethodName}Keys.all,
    queryFn: () =>
      request<{ {resolverMethodName}: {ResponseTypeName}[] }>(
        GRAPHQL_ENDPOINT,
        {RESOLVER_CONST}_QUERY,
        { filter }
      ).then(data => data.{resolverMethodName}),
  });
}
```

### 4. Mutation Hook (POST / PUT / PATCH / DELETE)
When the `operationId` maps to a mutating HTTP method, use `useMutation` instead:

```typescript
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '@/lib/api-client';

export interface {RequestTypeName} {
  [key: string]: unknown;
}

export function use{HookName}() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (payload: {RequestTypeName}) =>
      api.{httpMethod}<void>('{path}', payload),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['{relatedQueryKey}'] });
    },
  });
}
```

> **Related Query Key**: Derive from the resource noun in the path (e.g., path `/grades` → invalidate `['listGrades']` or the closest GET operationId key).

### 5. One File Per Unique `data_ref`
- Generate exactly **one stub file per unique `data_ref`** across all manifest nodes.
- If multiple nodes share the same `data_ref`, only one hook file is generated.
- File location: `src/hooks/use-{dataRef}.ts` (kebab-case filename).
- **Additional dependency for GraphQL hooks**: `graphql-request` must be installed alongside `@tanstack/react-query`. If not present, emit an installation note in the report.

---

## ⚠️ Output Requirements
- **Format**: TypeScript `.ts` files — no JSX.
- **Stubs**: Type bodies intentionally minimal (`[key: string]: unknown`). Downstream developers expand them.
- **No Invention**: Do not fabricate type fields not derivable from the OpenAPI `200` response schema.
- **No Registry Coupling**: Do NOT wire or import these hooks directly into standard presenter components in `component-registry.ts`. Presenter components must remain purely declarative. The generated hooks are intended to be consumed at the page entry/layout layer, custom data containers, or dynamic custom actions inside the UI spec context.
- **Dependencies Assumed**: `@tanstack/react-query` and `@/lib/api-client` must already be installed. If not present, emit an installation note in the workflow report.
