# US-04 個人化錯題追蹤 (Personalized Error Tracking)

## 背景 (Background)
透過演算法針對使用者的弱點進行記錄與排程，達到最高效率的單字記憶。

---

## US-04-01：錯題自動收錄與演算法排程

**身份**：系統 (System)

> **As a** 系統，  
> **I want to** 當使用者答錯題目時自動將該單字寫入「專屬錯題本」並增加錯誤權重，依據間隔重複演算法排定優先複習佇列，  
> **So that** 使用者能在遺忘曲線的關鍵點精準複習。

### 驗收條件 (Acceptance Criteria)
- **AC1 (事件紀錄)**：使用者每一次答錯，系統必須新增一筆「錯題紀錄 (ErrorEvent)」，保存該次答錯的具體測驗題目 (`QuizQuestion`)，作為錯題本歷史回顧使用。
- **AC2 (狀態更新)**：同時，系統必須非同步地更新該單字的「單字熟練度 (WordMastery)」狀態，將該單字的錯誤權重 (Error Weight) +1。每個使用者對特定單字只會有一筆熟練度狀態。
- **AC3 (演算法排程)**：系統應依據錯誤權重與上次複習時間，計算該單字的下次複習日期 (Next Review Date)，並更新於「單字熟練度 (WordMastery)」中。
- **AC4 (優先佇列)**：當使用者進入「錯題複習專區」時，系統應優先撈取 Next Review Date 為今日(或已過期)且錯誤權重最高的單字，並從其對應的歷史「錯題紀錄 (ErrorEvent)」中隨機挑選或重新派送題目。
- **AC5 (權重衰減與移除)**：當使用者在錯題複習專區成功答對該單字的題目時，系統應將該單字的錯誤權重 (Error Weight) 減 1。
  - 若權重減至 `0`：清空下次複習日期 (`Next Review Date = null`)，表示該單字已熟練，移出错題複習佇列。
  - 若權重仍 `> 0`：則將下次複習日期依演算法設定向後延遲（例如：`Today + 3` 天），確保未來再次複習。

## US-04-02：關鍵任務日誌 (Critical Mission Log)

**身份**：使用者 (Learner)

> **As a** 使用者，  
> **I want to** 從首頁進入「關鍵任務日誌」專區，透過時間範圍篩選並以單字卡切換的方式回顧我曾答錯的題目，  
> **So that** 我能在沒有測驗壓力的情況下，隨時針對自己的學習盲點進行專注的檢討。

### 驗收條件 (Acceptance Criteria)
- **AC1 (資料來源與時間篩選)**：此功能為獨立的純瀏覽模式，直接查詢 `ErrorEvent` 的歷史紀錄（視為 Severity Error Log）。畫面上方必須提供三個時間區間選項供切換，以限制資料量避免造成使用者焦慮：
  - 今日 (預設)
  - 過去一週
  - 過去一個月
- **AC2 (單字卡呈現模式)**：頁面不得採用條列或表格，必須採用「單字卡 (Card)」設計，一次僅呈現一筆 `ErrorEvent` 紀錄。使用者可透過「上一筆/下一筆」按鈕（或左右滑動）來切換查看。
- **AC3 (卡片顯示欄位)**：每張歷史紀錄卡片必須明確呈現以下欄位資訊：
  - **目標單字與發音**：巨大的英文單字，並附帶「喇叭圖示」，點擊可透過 TTS 無限次重聽發音。
  - **詞性與中文翻譯**：標示該單字的詞性 (如 noun, verb) 與對應的中文翻譯。
  - **錯誤對比 (案發現場)**：明確標示「正確答案是：[正解選項]」與「你當時選了：[使用者點擊的誘答項]」，直觀呈現當初的盲點。
  - **情境句與翻譯**：完整還原當時測驗的英文情境句 (Contextual Cloze)，並在下方附帶該情境句的中文翻譯。
  - **記憶輔助解析**：直接引用並顯示題庫中該單字的「字根字首拆解」與「聯想記憶法」兩個欄位資訊。
- **AC4 (無狀態影響)**：使用者在此專區的所有瀏覽與聽音行為，皆純屬回顧，不會改變底層 `WordMastery` 的演算法權重與排程狀態。
