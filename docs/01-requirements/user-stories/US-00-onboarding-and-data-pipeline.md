# US-00 系統冷啟動與資料管線 (Onboarding & Data Pipeline)

## 背景 (Background)
提供使用者初次進入系統時的學制選擇，並確保系統能提供毫秒級的題庫載入體驗，降低延遲感。

---

## US-00-01：首次登入與初始設定 (First Login & Onboarding)

**身份**：系統 (System) / 使用者 (Learner)

> **As a** 系統與使用者，  
> **I want to** 在使用者第一次成功登入時，引導選擇目標學制並自動建立使用者資料，  
> **So that** 使用者能以最少步驟完成註冊，且系統能立即派發符合其程度的題庫。

### 驗收條件 (Acceptance Criteria)
- **AC1 (身分驗證檢查)**：當使用者透過驗證機制 (如 OAuth / SSO) 登入時，系統應檢查 `user` 資料表。若已有資料，則直接進入首頁或上次學習進度。
- **AC2 (初始設定畫面)**：若查無該使用者（即首次登入），系統必須先顯示「目標學制」選擇畫面，選項至少包含「國中會考」與「高中學測」。
- **AC3 (自動註冊)**：使用者選定學制後，系統自動在資料庫內新增一筆該使用者的資料，並寫入其選擇的學制 (`TargetLevel`) 與預設角色 (`Learner`)。
- **AC4 (後續修改)**：使用者日後可以在設定頁面隨時更改目標學制。更改後，系統未來派送的題目應立即切換至新難度。



## US-00-02：官方詞彙表匯入與正規化 (Offline Script)

**身份**：資料工程師 (Data Engineer)

> **As a** 資料工程師，  
> **I want to** 透過本地端資料處理腳本 (Data Pipeline Scripts) 將大考中心 7000 字與國中會考 2000 字的官方詞彙表匯入開發資料庫並轉為 Seed Data，  
> **So that** 系統部署上線時，正式環境已擁有標準化、分級明確的單字種子庫 (Seed Words)。

### 驗收條件 (Acceptance Criteria)
- **AC1 (離線資料萃取 ETL)**：建立獨立的 Python/Node 腳本，從官方 PDF 擷取表格內容，並轉換為 Markdown (MD)、CSV 或 JSON 等結構化格式，不與 Web 後端 API 綁定。
- **AC2 (標籤與屬性)**：處理腳本必須正確標記每個單字的「學制」(國中/高中) 與「難度等級」(基礎/進階 或 Level 1~6)。
- **AC3 (防呆去重)**：腳本若遇到跨學制重複的單字（例如國中進階字與高中 Level 2 重疊），應能自動去重 (Deduplication) 或進行關聯整併。
- **AC4 (產出 Seeding 檔)**：腳本最終產出乾淨的 `V1__Seed_Vocabulary.sql` (或對應格式)，供 Backend 在應用程式啟動時透過 Flyway/Liquibase 自動寫入資料庫。

## US-00-03：AI 批次生成靜態題庫 (Offline Script)

**身份**：資料工程師 (Data Engineer)

> **As a** 資料工程師，  
> **I want to** 透過本地端自動化腳本，根據單字種子庫呼叫 AI API 批次生成對應的測驗題目，並打包成資料庫 Seeding 檔案，  
> **So that** Web 應用程式上線後，資料庫內已經備妥數萬題高品質的靜態題庫供使用者作答，後端無需實作繁瑣的 AI 呼叫邏輯。

### 驗收條件 (Acceptance Criteria)
- **AC1 (本地批次生成)**：建立獨立的腳本，能讀取正規化後的單字庫，並批次呼叫 AI 模型生成題目（包含情境句、中譯、1正解與3誘答項）。
- **AC2 (誘答項規則)**：生成的誘答項必須與正解詞性相同，且符合同一難度級別。AI 應優先選擇拼寫相似或語意相近的單字作為誘答項。
- **AC3 (格式與文法驗證)**：AI 生成情境句時，正解需包含詞類變化，且四個選項皆須符合該挖空處的文法狀態。腳本需進行格式驗證，確認無誤後才收錄。
- **AC4 (考頻決定題數)**：腳本應根據單字的「歷屆考頻 (ExamFrequency)」決定生成數量。高頻字 (4-5星) 生成 3~5 題；低頻字 (1-2星) 生成 1 題。若該單字已達題數上限，腳本執行時自動略過。
- **AC5 (產出 Seeding 檔)**：腳本最終將生成的題目轉換為 `V2__Seed_Quizzes.sql` (或 JSON Seeding 檔)，由後端伺服器在部署或啟動時一併載入。
