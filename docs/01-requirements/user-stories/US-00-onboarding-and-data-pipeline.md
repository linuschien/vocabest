# US-00 系統冷啟動與資料管線 (Onboarding & Data Pipeline)

## 背景 (Background)
提供使用者初次進入系統時的學制選擇，並確保系統能提供毫秒級的題庫載入體驗，降低延遲感。

---

## US-00-01：動態難度分級選擇

**身份**：使用者 (Learner)

> **As a** 使用者，  
> **I want to** 在首次登入或設定頁面中選擇目標學制（國中會考 1200/2000字，或高中學測/頂標 7000字），  
> **So that** 系統能推播符合我當前程度的題庫。

### 驗收條件 (Acceptance Criteria)
- **AC1**：使用者首次啟動系統時，必須顯示「目標學制」選擇畫面，選項至少包含「國中會考」與「高中學測」。
- **AC2**：選擇後，系統應記錄此設定，未來產出的題目皆從對應級別之單字庫中抽取。
- **AC3**：使用者可以在設定頁面隨時更改目標學制。更改後，新產出的題目應立即切換難度。

---

## US-00-02：靜態題庫派送

**身份**：系統 (System)

> **As a** 系統，  
> **I want to** 從預先由 AI 生成好的關聯式資料庫中抽取題目（包含英文情境句、中文翻譯、1正解與3誘答項），  
> **So that** 確保前端介面能在毫秒級載入，避免即時生成帶來的延遲。

### 驗收條件 (Acceptance Criteria)
- **AC1**：題目必須從本地端或遠端靜態資料庫取得，不依賴即時的 AI 生成 API。
- **AC2**：抽取的題目資料結構必須完整，包含: 挖空的英文句子、完整的中文翻譯、1個正確選項及3個錯誤(誘答)選項。
- **AC3**：請求一批題目的 API 響應時間必須小於 200 毫秒（Happy Path）。
