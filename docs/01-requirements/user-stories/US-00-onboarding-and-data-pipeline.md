# US-00 系統冷啟動與資料管線 (Onboarding & Data Pipeline)

## 背景 (Background)
提供使用者初次進入系統時的學制選擇，並確保系統能提供毫秒級的題庫載入體驗，降低延遲感。

---

## US-00-01：首次登入與初始設定 (First Login & Onboarding)

**身份**：系統 (System) / 使用者 (Learner)

> **As a** 系統與使用者，  
> **I want to** 在使用者第一次成功登入時，引導選擇目標學制並自動建立使用者資料，  
> **So that** 使用者能以最少步驟完成註冊，且系統能立即派發符合其程度的題庫。

### 驗收條件 (Acceptance Criteria)
- **AC1 (身分驗證檢查)**：當使用者透過驗證機制 (如 OAuth / SSO) 登入時，系統應檢查 `user` 資料表。若已有資料，則直接進入首頁或上次學習進度。
- **AC2 (初始設定畫面)**：若查無該使用者（即首次登入），系統必須先顯示「目標學制」選擇畫面，選項至少包含「國中會考」與「高中學測」。
- **AC3 (自動註冊)**：使用者選定學制後，系統自動在資料庫內新增一筆該使用者的資料，並寫入其選擇的學制 (`TargetLevel`) 與預設角色 (`Learner`)。
- **AC4 (後續修改)**：使用者日後可以在設定頁面隨時更改目標學制。更改後，系統未來派送的題目應立即切換至新難度。



## US-00-02：官方詞彙表匯入與正規化

**身份**：資料工程師 / 系統管理員 (Data Engineer / Admin)

> **As a** 系統管理員，  
> **I want to** 將大考中心 7000 字與國中會考 2000 字的官方詞彙表匯入系統資料庫，  
> **So that** 系統擁有標準化、分級明確的單字種子庫 (Seed Words) 供後續產生題目使用。

### 驗收條件 (Acceptance Criteria)
- **AC1 (資料萃取與清洗 ETL)**：由於原始官方資料為 PDF 格式，需具備一次性的資料處理機制（可透過 MarkItDown 類型的轉換工具或自訂腳本），能從 PDF 擷取表格內容，並轉換為 Markdown (MD)、CSV 或 JSON 等易於機器讀取的結構化格式。
- **AC2 (結構化匯入)**：系統應支援將清洗後的結構化資料 (MD/CSV/JSON) 匯入資料庫。特別是支援解析 Markdown 表格，以利直接相容現代 AI 萃取工具的產出結果。
- **AC3 (標籤與屬性)**：匯入時必須正確標記每個單字的「學制」(國中/高中) 與「難度等級」(基礎/進階 或 Level 1~6)。
- **AC4 (防呆去重)**：若遇到跨學制重複的單字（例如國中進階字與高中 Level 2 重疊），系統應能自動去重 (Deduplication) 或進行關聯整併。
- **AC5 (後台觸發與非同步處理)**：此匯入流程應可由系統管理員於後台介面 (US-05-02) 手動觸發（如上傳檔案）。因處理耗時，系統需以非同步背景任務 (Asynchronous Background Job) 執行，並提供執行進度與狀態（如：處理中 50%、完成、失敗）。

## US-00-03：AI 批次生成靜態題庫

**身份**：系統管理員 (Admin) / 系統 (System)

> **As a** 系統，  
> **I want to** 根據資料庫中的單字種子庫，透過呼叫 AI API 批次生成對應的測驗題目（情境句、中譯、誘答項），並存入資料庫，  
> **So that** 應用程式上線後，使用者隨時有足夠的靜態題庫可以作答，不須等待即時生成。

### 驗收條件 (Acceptance Criteria)
- **AC1 (基本生成)**：系統應具備批次生成腳本，能遍歷單字庫並呼叫 AI 模型生成題目。
- **AC2 (誘答項規則)**：生成的誘答項 (Distractors) 必須與正解詞性相同，且符合同一難度級別。針對易混淆字，AI 應優先選擇拼寫相似 (如 adapt/adopt) 或語意相近的單字作為誘答項。
- **AC3 (格式驗證)**：每批次生成的題目必須經過格式驗證（包含挖空底線、四個選項不重複）才能寫入靜態題庫資料庫。
- **AC4 (考頻決定題數)**：批次生成腳本應根據單字的「歷屆考頻 (ExamFrequency)」決定生成的題目數量。例如：高頻字 (4-5星) 需生成 3~5 題不同情境的題目；低頻字 (1-2星) 生成 1 題即可。
- **AC5 (詞類變化與文法相容)**：AI 生成情境句時，正解不應僅限於單字原型。必須包含詞類變化（如過去式、複數型、比較級），且四個選項皆須符合該挖空處的文法狀態（例如正解為 decided，則誘答項也必須是過去式動詞）。
- **AC6 (後台觸發與進度追蹤)**：此任務為單次觸發 (On-Demand) 的背景工作，無自動排程 (No Scheduler)。管理員可於後台手動觸發，並於介面檢視即時進度（如：已生成 150 / 2000 題）。
- **AC7 (數量上限與防呆)**：執行批次生成時，系統應檢查資料庫中該單字已存在的題目數量。若已達預期題數（如低頻字 1 題、高頻字 3 題），則自動略過該單字，避免產生過多類似且冗餘的題目。
