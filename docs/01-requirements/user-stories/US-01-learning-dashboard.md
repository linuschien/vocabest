# US-01 學習儀表板與首頁 (Learning Dashboard)

## 背景 (Background)
首頁是使用者每天登入看到的第一個畫面。設計目標為「無痛啟動」，透過顯著的進度追蹤與捷徑，降低學習的心理門檻。

---

## US-01-01：一鍵啟動測驗

**身份**：使用者 (Learner)

> **As a** 使用者，  
> **I want to** 在首頁看到巨大的「開始今日任務」按鈕，  
> **So that** 點擊後能跳過所有設定，直接進入該級別的測驗心流。

### 驗收條件 (Acceptance Criteria)
- **AC1**：首頁必須有一個明顯(大尺寸、高對比)的主按鈕。
- **AC2**：點擊按鈕後，立即轉場至測驗畫面，無需任何中間設定或確認對話框。
- **AC3**：若當日已有未完成的測驗進度，點擊後應接續上一次的進度繼續。

---

## US-01-02：進度與連勝追蹤 (微型成就感)

**身份**：使用者 (Learner)

> **As a** 使用者，  
> **I want to** 在首頁清楚看到「今日完成題數進度條」與「連續學習天數（火焰圖標）」，  
> **So that** 利用沉沒成本心理激勵我每天登入。

### 驗收條件 (Acceptance Criteria)
- **AC1**：首頁必須顯示「今日完成題數」與「每日目標題數」的進度條 (Progress Bar)。
- **AC2**：首頁必須顯示火焰圖標與對應的數字，代表連續登入並完成至少一題的天數。
- **AC3**：若跨越午夜(系統預設時區 00:00) 且前一日未完成任何題目，連續天數將歸零重計。

---

## US-01-03：錯題複習專區入口

**身份**：使用者 (Learner)

> **As a** 使用者，  
> **I want to** 在首頁能看見「今日待複習錯題數量」，並能一鍵進入專屬的錯題特訓模式，  
> **So that** 針對即將遺忘的單字進行精準打擊。

### 驗收條件 (Acceptance Criteria)
- **AC1**：若當日錯題佇列中有大於 0 的題目，首頁必須顯示「待複習錯題: X 題」的提示與入口按鈕。
- **AC2**：若錯題佇列為 0，則顯示「太棒了！今日無錯題待複習」或將按鈕反灰。
- **AC3**：點擊入口後，進入特殊測驗模式，題目全數來自錯題佇列。
